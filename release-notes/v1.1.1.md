# ESP32RC v1.1.1 Release Notes

**Release Date:** 2026-03-07

## Summary

V1.1.1 adds improved code reliability with atomic RC reads and automatic safety recovery, plus comprehensive testing infrastructure and documentation.

**Build Status:** ✅ All environments compile successfully  
**Tests:** ✅ 12/12 sanity checks pass  
**Hardware:** Tested on ESP32-DEV with Adafruit INA219 sensor

## What's New

### 🛡️ Reliability Improvements

**Atomic RC Read Helpers**
- Added thread-safe volatile access for RC channel values
- Prevents race conditions between ISR and main loop
- Applies throughout codebase: power control, lights, auxiliary outputs

**Automatic Safety Recovery**
- System now recovers from safety shutdown after 5-second stabilization
- Monitors overcurrent, voltage violations continuously  
- Only recovers when conditions are safe (current < threshold-2A, voltage in range)
- Prevents permanent lockout situations

### 🧪 Testing & Verification

**Automated Build Tests**
- 12 comprehensive sanity checks verify:
  - Both PlatformIO environments build successfully
  - Configuration constants defined correctly
  - Critical functions present in firmware
  - Atomic helpers implemented
  - Recovery mechanism integrated
- Run locally: `python scripts/build_sanity_tests.py`

**Interactive Hardware Verification**
- Serial-based hardware testing via `hardware_verification.py`
- Checks firmware boot, system status, INA219 sensor
- Tests self-test mode functionality
- Usage: `python scripts/hardware_verification.py COM3 115200`

## Compatibility

- **ESP32 Board:** ESP32-DEV Module (ESP32 WROOM)
- **Framework:** Arduino 3.20017.241212+
- **Required Libraries:** Adafruit INA219 v1.2.1, Adafruit BusIO v1.14.1
- **Voltage Range:** 10-26V (with stock INA219)
- **Current Range:** Up to 50A (with appropriate MOSFET/shunt)

## Migration from v1.1.0

**No breaking changes.** Existing hardware setups require no changes:

1. Download latest firmware binary (from GitHub Releases)
2. Upload via PlatformIO: `pio run --target upload`
3. System will automatically recover if previously in shutdown state
4. Current scale settings preserved from NVS

## Known Issues

See `docs/PROJECT_NOTES.md` for detailed list:
- ISR timing precision (acceptable, no action needed)
- Global current scaling (documented, tunable)
- No manual recovery command in self-test (auto-recovery is safer)
- Watchdog timer not enabled (intentional, allows graceful RC failsafe)
- Serial monitor frequency fixed at compile time (can be adjusted in config)
