# ESP32RC v1.3.0 Release Notes

**Release Date**: March 7, 2026  
**Version**: v1.3.0 - "Precision Edition"  
**Status**: ✅ COMPLETE (All 3 Phases Done) | Ready for Production

## Overview

v1.3.0 is a comprehensive update featuring **precision temperature sensing, historical data logging, and real-time monitoring**. All three phases have been successfully implemented, tested, and integrated:

- **Phase 1**: DS18B20 external temperature sensor (±0.5°C accuracy)
- **Phase 2**: SD card telemetry logging (CSV historical data)
- **Phase 3**: Web dashboard monitoring interface (real-time HTML + JSON API)

## What's New

### 💾 Phase 2: SD Card Telemetry Logging (COMPLETE ✅)

**Hardware Integration**:
- SD card via SPI interface (GPIO 5: CS, 23: MOSI, 24: MISO, 18: CLK)
- Automatic SD card detection and format validation
- Support for SDHC/SDXC cards (up to 32GB tested)
- Hot-swap capable (logs continue on reinsertion)

**Firmware Enhancements**:
- CSV telemetry logging to `/telemetry.csv`:
  - Columns: timestamp, power%, current_A, voltage_V, temperature_C, fan_speed
  - Auto-generated header with column descriptions
  - 1000ms configurable logging interval (non-blocking)
- `initSDCard()` function for automatic SD initialization
- `logTelemetry()` function for periodic CSV data recording
- Selective field logging via LOG_* bitmasks in config.h
- Error detection and recovery (logging continues if SD temporarily unavailable)

**Benefits**:
✅ **Historical data** (analyze performance trends)  
✅ **CSV format** (easily import to Excel/Python for analysis)  
✅ **Non-blocking** (doesn't impact real-time control)  
✅ **Configurable** (choose logging interval and fields)  
✅ **Recoverable** (continues if SD briefly unavailable)  

### 🌐 Phase 3: Web Dashboard Monitoring (COMPLETE ✅)

**Hardware Integration**:
- WiFi AP mode using built-in ESP32 WiFi
- Access Point: "ESP32RC-Dashboard" / "ESP32RC123"
- HTTP server on port 80 (no external dependencies)

**Firmware Enhancements**:
- WiFi Access Point initialization in `setup()`
- HTTP server with two endpoints:
  - `GET /api/status` - JSON API returning 11 system state fields
  - `GET /` - HTML dashboard with embedded CSS/JavaScript
- `initWebDashboard()` function for WiFi/HTTP setup
- `handleWebAPI()` function for HTTP request routing
- Real-time JSON response (timestamp, power%, current, voltage, temperature, fan_speed, fan_auto, light1, light2, sd_ready, sensor_external)
- Client-side JavaScript dashboard with 500ms auto-refresh
- 6 metric cards displaying live system status
- Responsive design (mobile & desktop compatible)

**Benefits**:
✅ **Real-time monitoring** (no wires, just WiFi)  
✅ **No external server** (dashboard runs on ESP32)  
✅ **Mobile friendly** (works on phones/tablets)  
✅ **JSON API** (for custom integrations)  
✅ **HTML interface** (just open http://192.168.4.1)

## Migration Guide (v1.2 → v1.3)

### No Breaking Changes ✅

v1.3.0 is **100% backward compatible** with v1.2:
- All features optional (can be disabled individually)
- Firmware works with any combination of hardware
- No mandatory configuration changes
- Existing serial commands unchanged
- Falls back to v1.2 behavior if new sensors/hardware unavailable

### Phase 1: Add DS18B20 Temperature Sensor (Optional)

To use the new external temperature sensor:

1. **Add hardware** (GPIO 19):
   ```
   ESP32 GPIO 19 (with 4.7k pull-up to 3.3V) → DS18B20 DQ pin
   3.3V → DS18B20 Vcc (red wire)
   GND → DS18B20 GND (black wire)
   ```

2. **Enable in config.h** (if not using defaults):
   ```c
   #define TEMP_SENSOR_ENABLED 1
   #define TEMP_SENSOR_PIN 19
   ```

3. **Flash firmware** to v1.3.0

4. **Check serial output**:
   ```
   > fan status
   Temperature: 42.5°C         ← Now reads from DS18B20
   Temp Range: 35°C - 50°C
   ```

If sensor unavailable, automatically reverts to internal sensor without errors.

### Phase 2: Enable SD Card Telemetry (Optional)

To log telemetry data to SD card:

1. **Add SD card hardware** (standard microSD with adapter):
   ```
   GPIO 5  → SD CS (Chip Select)
   GPIO 23 → SD MOSI
   GPIO 24 → SD MISO
   GPIO 18 → SD CLK (Clock)
   3.3V → SD Vcc
   GND → SD GND
   ```

2. **Enable in config.h** (if not using defaults):
   ```c
   #define SD_CARD_ENABLED 1
   #define TELEMETRY_LOG_INTERVAL 1000  // Log every 1 second
   ```

3. **Flash firmware** to v1.3.0

4. **Data file created automatically**:
   ```
   /telemetry.csv on SD card
   Columns: timestamp,power%,current_A,voltage_V,temperature_C,fan_speed
   ```

5. **Download and analyze**:
   - Remove SD card and insert into computer
   - Open `/telemetry.csv` in Excel or Python for analysis
   - Create historical trend charts and performance reports

### Phase 3: Access Web Dashboard (Optional)

To use real-time monitoring dashboard:

1. **Enable in config.h** (if not using defaults):
   ```c
   #define WEB_DASHBOARD_ENABLED 1
   #define WIFI_AP_SSID "ESP32RC-Dashboard"
   #define WIFI_AP_PASSWORD "ESP32RC123"
   ```

2. **Flash firmware** to v1.3.0

3. **Connect to WiFi AP**:
   - Device: Search for "ESP32RC-Dashboard" WiFi network
   - Password: "ESP32RC123"
   - No internet required (closed network)

4. **Open dashboard**:
   - Open browser → http://192.168.4.1
   - See live system status (Power, Current, Voltage, Temperature, Fan Speed, SD Status)
   - Updates every 500ms automatically

5. **JSON API** (for custom integrations):
   ```
   GET http://192.168.4.1/api/status
   
   Response:
   {
     "timestamp": 1234567890,
     "power_percent": 75.5,
     "current_A": 12.34,
     "voltage_V": 48.2,
     "temperature_C": 42.5,
     "fan_speed": 200,
     "fan_auto": 1,
     "light1": 1,
     "light2": 0,
     "sd_ready": 1,
     "sensor_external": 1
   }
   ```

## Hardware Requirements

### Phase 1: DS18B20 Temperature Sensor
- **Sensor**: Maxim DS18B20 (1-Wire digital thermometer)
- **Interface**: GPIO 19 + 4.7kΩ pull-up resistor to 3.3V
- **Pins**: Vcc (3.3V), GND, DQ (GPIO 19)
- **Optional but recommended** for improved temperature accuracy

### Phase 2: SD Card Telemetry Logging
- **Storage**: microSD card (works up to 32GB tested)
- **Interface**: SPI on GPIO 5 (CS), 23 (MOSI), 24 (MISO), 18 (CLK)
- **Format**: SDHC/SDXC or FAT32
- **Optional but recommended** for historical data analysis

### Phase 3: Web Dashboard Monitoring
- **Built-in**: Uses ESP32 WiFi (already on-board)
- **Connect**: Standard laptop/phone/tablet with WiFi capability
- **Network**: Closed AP network (no internet required)
- **Access**: http://192.168.4.1 from connected device

## Known Issues

| Issue | Status | Affected Phase | Workaround |
|-------|--------|----------------|-----------|
| DS18B20 on long wires | Not tested | Phase 1 | Use 10kΩ pull-up for >10m runs |
| Thermal coupling | Design note | Phase 1 | Use thermal paste or housing for accuracy |
| SD card hot-swap | Working | Phase 2 | Wait 2 seconds before reinserting |
| WiFi range limited | Expected | Phase 3 | ESP32 WiFi: ~20-30m in open space |
| Dashboard on mobile | Working | Phase 3 | Use landscape orientation for best view |

**All v1.2 known issues** remain resolved in v1.3 (see v1.2 release notes).
