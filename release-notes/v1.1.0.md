# ESP32RC 1.1.0

This release hardens the runtime safety behavior of the active PlatformIO firmware and aligns the repository around that firmware as the source of truth.

## Highlights

- Safe startup behavior with no output before the first valid `CH1` pulse
- Throttle gated strictly by live `CH1` data
- Optional serial self-test firmware for bring-up without a live RC transmitter
- Runtime and persisted current-scale calibration using ESP32 `Preferences`
- Legacy sketches moved out of the repository root
- GitHub Actions build coverage for both PlatformIO environments

## Included changes

- Added `docs/BRINGUP.md` for first flash and first hardware power-up
- Added `env:esp32dev_selftest` for serial-controlled bring-up
- Added self-test serial commands for INA219 inspection and current-scale tuning
- Added persisted current-scale storage and reload across reboot
- Updated documentation to match the active PlatformIO firmware
- Fixed startup failsafe, `CH1` gating, ISR-shared timestamp handling, and missing-sensor polling
